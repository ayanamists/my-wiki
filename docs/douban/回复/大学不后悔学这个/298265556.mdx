笨人ip地某九在读，行政管理专业，突然发现我校计算机专业的转专业政策是面对全校本科生的

转专业考试由机试和面试组成，二者各占一半，机试科目及范围是：程序设计（C/C++/Java/Python）。

我以前高中的时候其实文理成绩差不多，因为一些意外学了文科，高中学习还好，但是大学学的真的很痛苦，感觉这不是我想象的文科。之前就在考虑辅修数学，后来又想转到法学，直到我今天突然看到可以转到计算机...

太激动了整篇文章有点语无伦次，我们大概是五月份考试，现在自学这些东西来得及吗，我们大一一点计算机都没学，只学了文科高数，大家对自学计算机有什么建议吗？计算机是不是对电脑也有要求啊，我电脑才三千多，是不是会带不动...不过我应该可以用图书馆的电脑。求求求求了大家给点建议吧

我们专业挺水的，下周有三门课结课，之后的课就更少了，我应该会有很多时间来自学的。不过转专业的竞争压力应该会非常大...omg虽然很害怕但是这也许是我最后一次能够接触理工科的机会了...

上面某些人的言论，只能证明她们学识的浅薄，这里我就不再长篇大论地批判了。
现在回到楼主的问题。我不太清楚你们学校的具体政策，不过需要提醒的是，我本科时候参与过转专业考试的监考，我发现他们里面有不少高中学信息竞赛（OI）而大学因为种种原因没就读计算机专业的人。我必须泼一下冷水：他们在上大学之前就有了相当程度的编程实践，短短几个月的学习经验是没法竞争过他们的。这一点需要楼主自己确认（可以问问去年的情况）。
我得讲个我真实的经历。大三下学期的时候，怀着对函数式编程的喜爱，我当上了我们学校
Haskell
编程课的助教。这门课是面向全校的公选课（我也问过老师为啥不开专业课，老师无奈地说大家可能对这个没兴趣，以前开过，选的人实在太少），里面有不少和你一样非计算机专业的同学。其中，我对一位木桶饭同学印象非常深。
这位同学是医学专业的，可想而知，他（她）的学业压力非常大。我从第一次课就注意到了她，因为她是唯一一个在这门课上用
Linux 平台写代码的人。通过和她的交流，我发现她对 Linux
的熟悉程度甚至比我们计算机专业的很多同学还要好。她每次课作业的代码写的虽然有点生涩，但是态度非常认真。要知道，我们这门课是不算绩点的，很多同学作业都是随便写写而已。她也是唯一问我关于这门语言里比较复杂的特性（Monad）的同学。
就读计算机专业的人是有限的，转专业的名额也是有限的。但是，像我提到的那位同学一样学会写程序、对自己手中的计算机有更深刻的理解、用计算机解决很多自己的问题，我觉得这其实是每个人都可以做到的。楼主也许不妨将这次考试看成一个自己与计算机科学接触的机会，自己观察一下这门学科到底有不有趣，到底美在哪里，丑在哪里......
至于怎么自学，前面的同学已经提到了
csdiy，这个是很好的资源。如果问我的话，那我还是推荐学一门函数式语言，现在我会把用
Ocaml 的 CS3110\[1\]
作为首选。但是，考虑到准备考试的现实要求，那可能还是基于 Python
的某些课程（比如 cs61a\[2\] 和南大的
SICP\[3\]）比较合适。编程需要的是不断练习，无论学什么课程，一定得把习题完成。

> 现在回到楼主的问题。我不太清楚你们学校的具体政策，不过需要提醒的是，我本科时候参与过转专业考试的监考，我发现他们里面有不少高中学信息竞赛（OI）而大学因为种种原因没就读计算机专业的人。我必须泼一下冷水：他们在上大学之前就有了相当程度的编程实践，短短几个月的学习经验是没法竞争过他们的。这一点需要楼主自己确认（可以问问去年的情况）。
> 我得讲个我真实的经历。大三下学期的时候，怀着对函数式编程的喜爱，我当上了我们学校
> Haskell
> 编程课的助教。这门课是面向全校的公选课（我也问过老师为啥不开专业课，老师无奈地说大家可能对这个没兴趣，以前开过，选的人实在太少），里面有不少和你一样非计算机专业的同学。其中，我对一位木桶饭同学印象非常深。
> 这位同学是医学专业的，可想而知，他（她）的学业压力非常大。我从第一次课就注意到了她，因为她是唯一一个在这门课上用
> Linux 平台写代码的人。通过和她的交流，我发现她对 Linux
> 的熟悉程度甚至比我们计算机专业的很多同学还要好。她每次课作业的代码写的虽然有点生涩，但是态度非常认真。要知道，我们这门课是不算绩点的，很多同学作业都是随便写写而已。她也是唯一问我关于这门语言里比较复杂的特性（Monad）的同学。
> 就读计算机专业的人是有限的，转专业的名额也是有限的。但是，像我提到的那位同学一样学会写程序、对自己手中的计算机有更深刻的理解、用计算机解决很多自己的问题，我觉得这其实是每个人都可以做到的。楼主也许不妨将这次考试看成一个自己与计算机科学接触的机会，自己观察一下这门学科到底有不有趣，到底美在哪里，丑在哪里......
> 至于怎么自学，前面的同学已经提到了
> csdiy，这个是很好的资源。如果问我的话，那我还是推荐学一门函数式语言，现在我会把用
> Ocaml 的 CS3110\[1\]
> 作为首选。但是，考虑到准备考试的现实要求，那可能还是基于 Python
> 的某些课程（比如 cs61a\[2\] 和南大的
> SICP\[3\]）比较合适。编程需要的是不断练习，无论学什么课程，一定得把习题完成。
